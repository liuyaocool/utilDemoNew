#### 模型

- AP模型	**优选方案**
  - 实时性不高，高可用。
  - 一个不通，大不了换一个。
  - 客户端拉取服务列表，自己挑
- CP模型
  - 可用性不高。
  - 三级缓存不保证C
  - 集群间同步不保证C

#### 两种方式

- 多服务端之间相互调用 拉取，客户端注册一个服务就行
  - eureka.client.service-url.defaultZone=addr1,addr2,...
- 多服务端之间不通信，相互独立，客户端需要注册每一个服务
  - 客户端请求数据的时候会从多个服务上获取

#### 优化

1. 配置顺序
   - srv1：default-url=1,2,3
   - srv2：default-url=2,3,1
